## First-Run Survival Framework for Directive 8020

You don’t need advanced mastery on day one. You need a repeatable method. Use this four-phase loop whenever a new area opens.

### The 4-Phase Loop

1. **Survey**
   - Check exits, interactables, and elevation changes.
   - Identify fallback routes before advancing.

2. **Stabilize**
   - Complete low-risk tasks first (simple interactions, nearby pickups).
   - Avoid triggering multiple unknown events at once.

3. **Probe**
   - Advance a little, gather information, retreat if needed.
   - Confirm whether threats are scripted, roaming, or reactive.

4. **Commit**
   - Execute objective path only after understanding risk zones.
   - Save resources for uncertain encounters.

| Phase | Common Player Mistake | Better Approach |
|---|---|---|
| **Survey** | Sprinting forward immediately | Map the room and exits first |
| **Stabilize** | Ignoring easy prep interactions | Reduce chaos before major actions |
| **Probe** | Treating every sound as immediate attack | Validate pattern, then react |
| **Commit** | Overusing tools too early | Preserve key resources for real spikes |
